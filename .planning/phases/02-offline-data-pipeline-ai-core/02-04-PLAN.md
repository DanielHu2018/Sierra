<objective>
Write and run the friction scoring pipeline script (`3-score-friction.ts`) and the pipeline orchestrator (`run-pipeline.ts`). The scoring script loads `graph.json` and the RAG index, batches nodes ~20–25 at a time, retrieves the top 2–3 regulatory excerpts per batch via cosine similarity, calls Claude with structured JSON output to get `{ nodeId, frictionScore, justification }` per node, and writes `public/data/friction_cache.json`. The orchestrator ties all three pipeline steps together with disk-checkpoint skip logic so any single step can be re-run independently.

Purpose: `friction_cache.json` is the pre-computed AI artifact that powers ROUTE-04 (loaded at Phase 3 startup into Zustand store), HEAT-02 (heatmap layer), and HOVER-01/02 (segment justification popups). This is the last Phase 2 deliverable before Phase 3 can begin.
Output: `public/data/friction_cache.json` — all node IDs as keys, each with frictionScore and justification.
</objective>

Create `server/src/pipeline/3-score-friction.ts` as a standalone ESM TypeScript script.

**Required output contract:**
- `friction_cache.json` is an object (not array) keyed by node ID: `{ [nodeId: string]: FrictionEntry }`
- Each FrictionEntry has frictionScore (0–1 float) and justification (string) — nothing else
- No lat/lng/coordinate fields in any entry (AI-03)
- Justifications reference statute names, section numbers, and Texas location names (e.g., "per ESA Section 7, this node in Reeves County overlaps critical habitat for the Trans-Pecos spotted bat, adding approximately 8–12 months for formal USFWS consultation")
- Partial progress written to `server/data/friction_cache.partial.json` after each batch completes
- On script resume, skip batches whose node IDs already appear in the partial file
- All node IDs from graph.json appear in the final friction_cache.json (no missing nodes)
- frictionScore values are distributed across the 0–1 range (not all the same value)

**Script structure:**

1. **Load inputs:**
   - `graph.json` → array of GraphNode (from `public/data/graph.json` relative to repo root)
   - `regulations-embedded.json` → call `loadRAGIndex(path)` from `../rag/ragIndex.js`
   - `node-flags.json` (if it exists at `server/data/node-flags.json`) → `Map<string, NodeFlags>` for overlay context

2. **Load partial progress:**
   ```typescript
   const partialPath = path.resolve('./server/data/friction_cache.partial.json');
   const scored: Record<string, FrictionEntry> = existsSync(partialPath)
     ? JSON.parse(readFileSync(partialPath, 'utf-8'))
     : {};
   const remainingNodes = nodes.filter(n => !scored[n.id]);
   ```

3. **Batch remaining nodes into groups of ~20–25** (geographically nearby — slice by latitude row order so batch members are spatially coherent):
   ```typescript
   const BATCH_SIZE = 22;
   const batches: GraphNode[][] = [];
   for (let i = 0; i < remainingNodes.length; i += BATCH_SIZE) {
     batches.push(remainingNodes.slice(i, i + BATCH_SIZE));
   }
   ```

4. **Per batch — build Claude prompt:**
   - Include: node IDs, lat/lng coordinates, overlay flags (esaHabitat, privateLand, nearErcotCorridor, topoElevationM) if node-flags.json is available
   - RAG retrieval: embed a short location description ("Texas transmission corridor nodes in lat range X–Y, lng range A–B") using OpenAI embeddings, then call `retrieveTopK(embedding, 3)` to get the most relevant regulatory excerpts for that geographic area
   - Include the top-k regulation text excerpts in the prompt system message
   - Prompt instruction: "Score each node's friction for transmission line routing (0=no obstacles, 1=maximum friction). Base scores on the regulatory context provided. Return ONLY the JSON schema. Do NOT generate or modify any coordinates. Justify each score with specific statute references and Texas location names."

5. **Claude call:** Use structured output schema (Pattern 5). Parse `response.content[0].text` as JSON, extract `.scores` array. Validate each entry has the correct nodeId.

6. **Accumulate + write partial progress after each batch:**
   ```typescript
   for (const score of scores) {
     scored[score.nodeId] = {
       frictionScore: score.frictionScore,
       justification: score.justification,
     };
   }
   writeFileSync(partialPath, JSON.stringify(scored, null, 2));
   console.log(`Progress: ${Object.keys(scored).length}/${nodes.length} nodes scored`);
   ```

7. **Concurrency:** Wrap each batch call in `limit(() => withRetry(() => scoreBatch(...)))`. `pLimit(4)` limits to 4 concurrent Claude calls.

8. **Final output:** After all batches complete, verify all node IDs are present. Write to `public/data/friction_cache.json`. Optionally delete the partial file.
   ```typescript
   const missing = nodes.filter(n => !scored[n.id]);
   if (missing.length > 0) {
     throw new Error(`${missing.length} nodes missing from friction_cache. Re-run to resume.`);
   }
   writeFileSync(path.resolve('./public/data/friction_cache.json'), JSON.stringify(scored, null, 2));
   console.log(`friction_cache.json written with ${Object.keys(scored).length} entries`);
   ```

```typescript
import { existsSync } from 'fs';
import path from 'path';

const ROOT = path.resolve('./');

async function main() {
  console.log('Sierra Pipeline — starting');

  // Step 1: Scrape + embed regulations
  const ragPath = path.join(ROOT, 'server/data/regulations-embedded.json');
  if (existsSync(ragPath)) {
    console.log('Step 1: regulations-embedded.json exists — skipping scrape + embed');
  } else {
    console.log('Step 1: Scraping and embedding regulations...');
    await import('./1-scrape-embed.js');
  }

  // Step 2: Build graph
  const graphPath = path.join(ROOT, 'public/data/graph.json');
  if (existsSync(graphPath)) {
    console.log('Step 2: graph.json exists — skipping graph build');
  } else {
    console.log('Step 2: Building routing graph...');
    await import('./2-build-graph.js');
  }

  // Step 3: Score friction
  const frictionPath = path.join(ROOT, 'public/data/friction_cache.json');
  if (existsSync(frictionPath)) {
    console.log('Step 3: friction_cache.json exists — skipping friction scoring');
  } else {
    console.log('Step 3: Scoring node friction with Claude...');
    await import('./3-score-friction.js');
  }

  console.log('Pipeline complete. Artifacts ready for Phase 3.');
  console.log(`  graph.json: ${graphPath}`);
  console.log(`  friction_cache.json: ${frictionPath}`);
  console.log(`  regulations-embedded.json: ${ragPath}`);
}

main().catch(err => {
  console.error('Pipeline failed:', err.message);
  process.exit(1);
});
```
