### Gaps Summary

**1 gap blocking full AI-01 goal achievement:**

The `regulations-embedded.json` file on disk was generated by `generate-mock-data.ts` using a deterministic LCG hash — not by the real OpenAI `text-embedding-3-small` embedding model. All structural requirements (30 chunks, 1536 dimensions, 6 statutes, unit vectors, no external DB, loads at startup) are met. However, cosine similarity retrieval via `retrieveTopK` will return arbitrary chunks for any query because the embedding space carries no semantic meaning.

**Impact on downstream phases:** When Phase 3's `3-score-friction.ts` runs with real API keys, it will embed location descriptions and retrieve "top K" regulatory chunks — but those chunks will be random, not the most relevant statutes for that geographic area. Friction justifications will still cite statutes (because they appear in the chunk text), but regulatory context matching is broken.

**Fix is one command:** `OPENAI_API_KEY=<key> npx tsx server/src/pipeline/1-scrape-embed.ts` from repo root. The script is fully implemented, tested, and correct — it just needs to run with a real key.
