Create `server/src/pipeline/2-build-graph.ts` as a standalone ESM TypeScript script.

**Required output contract:**
- GraphNode schema: `{ id: string, lat: number, lng: number, neighbors: string[] }` — no overlay flags in output
- Grid spacing: ~25km; lat step = 25/111 degrees; lng step varies per latitude row (`25 / (111 * cos(lat * PI / 180))`)
- Node IDs: `${lat.toFixed(3)}_${lng.toFixed(3)}`
- 8-connected grid: each node connects to up to 8 geographic neighbors found by proximity search within `SPACING_KM * 1.5` distance (not by ID computation — avoids rounding issues per Pitfall 7)
- Overlay enrichment (in-memory only, NOT written to graph.json): esaHabitat, privateLand, nearErcotCorridor, topoElevationM
- BFS check: if reachability < 0.95, throw an error and do NOT write graph.json
- Write to `public/data/graph.json` only after BFS passes

**Script structure:**
1. Load all four GeoJSON overlay files from `public/data/` using `fs.readFileSync` + `JSON.parse`
2. Build the lat/lng grid: iterate lat from 25.84 to 36.50 with LAT_STEP, compute lngStep per row, generate all nodes with IDs
3. Enrich each node: run turf checks to produce `NodeFlags { esaHabitat, privateLand, nearErcotCorridor, topoElevationM }` — store enrichment flags in a separate `Map<string, NodeFlags>` keyed by node ID. Do NOT add flags to the GraphNode objects.
4. Build neighbor lists: for each node, find all other nodes within `SPACING_KM * 1.5` km using `turf.distance`. Push matching node IDs into `node.neighbors`. Cap at 8 neighbors (closest first).
5. Run BFS connectivity check (Pattern 4 from RESEARCH.md). If `visited.size / nodes.length < 0.95`, throw `new Error('BFS check failed: graph is not sufficiently connected')`.
6. Write `JSON.stringify(nodes)` to `../../public/data/graph.json` relative to the script file. Confirm the path resolves correctly using `path.resolve`.
7. Optionally write the enrichment flags map to `server/data/node-flags.json` for use by step 3 (friction scoring). This avoids re-computing turf checks in Plan 04.

**Key implementation notes:**
- ERCOT overlay: `ercot-grid.geojson` uses LineString geometry. Use `turf.nearestPointOnLine(line, pt)` for each feature, then check if `turf.distance(pt, nearestPt)` < 10 km. Set `nearErcotCorridor = true` if any line is within threshold.
- Wildlife/land overlays: use `turf.booleanPointInPolygon(pt, feature)`. Handle both Polygon and MultiPolygon geometry types.
- Topography: extract `feature.properties.elevation` (or similar property) for the nearest contour feature. Use `turf.nearestPoint` to find the closest contour node.
- Progress logging: log every 50 nodes during enrichment so the operator can see the pipeline isn't frozen.
- The script is invoked via `npx tsx server/src/pipeline/2-build-graph.ts` from the repository root (or via `npm run pipeline` in Plan 04's orchestrator).
